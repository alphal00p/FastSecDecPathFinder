# Universal Formula Cache

This note records the current cache-warming workflow for topology-independent
FSD projector formulae.  The cache lives under `cache/subtraction_formulae` and
is intentionally suitable for distribution outside the git repository.

The warmed universal pieces are:

- endpoint-projector formulae;
- regular Taylor source formulae;
- mapped-derivative chain-rule formulae.

The topology-specific prepared bundle and the experimental two-stage sector
evaluators are not counted as universal cache assets.

## CLI Mode

The cache warmer is exposed as a normal FSD subcommand:

```sh
.venv/bin/python FSD.py cache \
  --cache-loop-counts 1 2 \
  --cache-verify-samples-per-sector 8 \
  --cache-report-path docs/universal_cache_report.json \
  --cache-workdir .cache_warm \
  --no-progress
```

`--cache-cases` can be used instead of `--cache-loop-counts` to warm a selected
set of examples.  The verification step runs democratic sampling with equal
statistics in every sector and records the min/max sector timing.

All 1L and 2L DOT examples completed the verification pass.  The slowest
verified sector was `double_box` sector `PSD70`, at about
`3.46 ms/sample` with a roughly balanced evaluator/Python profile.  This is
acceptable for the 2L cache path and provides a useful baseline for spotting
3L sectors that still fall back to Python-heavy assembly.

The IBP endpoint-lowering cache variant was also generated with:

```sh
.venv/bin/python FSD.py cache \
  --cache-loop-counts 1 2 \
  --cache-verify-samples-per-sector 4 \
  --cache-report-path docs/universal_cache_report_ibp.json \
  --cache-workdir .cache_warm_ibp \
  --ibp-reduce-to-log-endpoint \
  --direct-projector-cache-term-threshold 0 \
  --no-progress
```

It verified successfully on the same examples.  The double-box IBP pass wrote
14 endpoint-projector JSONs, 10 regular Taylor evaluator groups, and 29
chain-rule evaluator groups.  The slowest verified sector was `PSD69`, at
about `19.8 ms/sample` with only four samples per sector.  This is still a
low-stat timing probe, but it confirms that the IBP universal assets can be
generated and used by the DOT examples.

## 3L Cache Estimate

The 1L/2L timing extrapolation is a lower bound only.  The hard triple-box
families contain six-axis endpoint structures that are much larger than
anything generated by the 2L examples above.

The current report therefore carries two estimates:

| estimate basis | small 3L examples | triple-box hard six-axis family |
|---|---:|---:|
| optimistic lower bound from 2L new-formula rate | 54 s | 431 s |
| calibrated lower bound using 90 s per hard source group | 6.25 h serial | 50 h serial |
| calibrated direct-formula bound using 823 s per six-axis formula | 57.2 h serial | 457 h serial |

The calibrated numbers are based on earlier PSD2/triple-box studies in this
workspace: one hard source-expression group exceeded 90 seconds locally, and
one direct six-axis regular formula took 823 seconds.  They are not a claim
that every 3L signature is this hard; they are a cluster-sizing warning for the
universal six-axis family.

Because these cache entries are universal and written signature-by-signature,
the task is embarrassingly parallel.  On a 100-worker cluster, the calibrated
triple-box range becomes roughly 0.5 to 4.6 hours before IO and per-worker RAM
constraints.  The practical next step is to generate the 3L universal cache on
a memory-rich cluster, package `cache/subtraction_formulae` as
`FSD_cache.tar.gz`, and install it with:

```sh
./install.sh --cache-tar /path/to/FSD_cache.tar.gz
```
